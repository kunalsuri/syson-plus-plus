SysON++

Web-based SysML v2 Modelling Environment — developer setup guide

Java 21 Spring Boot Node 22 React + TypeScript PostgreSQL 15 Docker Windows PowerShell

⚡ Quick Start

🆕 First time on this machine

Full setup + build

Checks all dependencies, installs missing tools, and compiles the backend (~5–15 min on first run). 

.\scripts-spp\setup-dev.ps1
🔄 Every day after that

Start everything

Starts database, backend, and frontend in order. Assumes the project is already built. 

.\scripts-spp\start-dev.ps1

Stop everything

.\scripts-spp\stop-dev.ps1

Safe stop: only kills the SysON++ Java process and the Vite server. Does not touch IntelliJ, VS Code, or any other running application. The PostgreSQL container is paused — not deleted — so your data is preserved.

📋 Prerequisites

Open a PowerShell terminal and verify each tool is installed before running the scripts.

fnm (Node version manager) Any fnm --version winget install Schniz.fnm

⚠️
GitHub Personal Access Token required for first build.
Maven downloads Sirius Web packages from GitHub Package Registry. Before running setup-dev.ps1 for the first time, set your token:

$env:PASSWORD = 'your-github-pat-here'

Create a token at github.com → Settings → Developer settings → Personal access tokens. Enable the read:packages scope.

📁 What Each Script Does

scripts-spp\start-dev.ps1

Daily orchestrator. Starts all three services in order — database first, then backend, then frontend. Waits for each service to be ready before starting the next. Use this every day.

Use: every time you sit down to work.

scripts-spp\setup-dev.ps1

First-time setup. Checks and installs all dependencies, runs the Maven build (~5–15 min), installs npm packages, then starts all services. Use this once when setting up a new machine.

Use: once on a new machine, or after a major dependency change.

scripts-spp\stop-dev.ps1

Safe shutdown. Stops only the SysON++ Java process (matched by JAR name), the Vite server (matched by port 5173), and pauses the PostgreSQL container. Will not touch any other application.

Use: when you are done working.

scripts-spp\start-db.ps1

Database only. Starts the PostgreSQL Docker container and waits for it to be ready on port 5432.

Use: if only the database needs to be restarted.

scripts-spp\start-backend.ps1

Backend only. Starts the Spring Boot JAR as a background process. Logs go to %TEMP%\sysonpp-backend.log. Waits up to 120 seconds for port 8080.

Use: after a Java code change and rebuild.

scripts-spp\start-frontend.ps1

Frontend only. Activates fnm Node v22, then starts the Vite dev server via turbo run start. Logs go to %TEMP%\sysonpp-frontend.log. Waits up to 60 seconds for port 5173.

Use: after a TypeScript/React change when the backend is already running.

🔧 Troubleshooting

npm crashes with an EPERM error about admin-local\AppData ▾

This happens when Node.js was installed system-wide by a different Windows user (admin-local) via NVM4W. The fix is to use fnm, which manages Node per-user.

winget install Schniz.fnm
# Open a new terminal, then:
fnm install 22.16.0
fnm use 22.16.0
node --version   # should print v22.16.0

The scripts activate fnm automatically, so once fnm is installed you should not need to do this again.

Maven build fails — authentication error or package not found ▾

The Maven build downloads Sirius Web packages from GitHub Package Registry and requires a Personal Access Token (PAT).

# In your PowerShell session before running setup-dev.ps1:
$env:PASSWORD = 'ghp_your_token_here'

To avoid doing this every session, add it to your PowerShell profile: notepad $PROFILE

Docker daemon is not running ▾

Open Docker Desktop from the Start Menu and wait for the whale icon in the taskbar to stop animating. Then run the script again. If Docker is not installed:

winget install Docker.DockerDesktop

After installing Docker Desktop, you must restart your computer before it will work.

Backend JAR not found — build the project first ▾

The JAR is produced by the Maven build. If you cloned a fresh copy of the repo, or after a major version change, you must build first.

.\scripts-spp\setup-dev.ps1

This can take 5–15 minutes on the first run as it downloads all Maven dependencies.

Port 8080 or 5173 is already in use ▾

Another process is occupying the port. Run the stop script first, then try again.

.\scripts-spp\stop-dev.ps1
.\scripts-spp\start-dev.ps1

To find what process owns a port: Get-NetTCPConnection -LocalPort 8080 -State Listen

http://localhost:8080 shows "Page Not Found" ▾

That is expected. Port 8080 is the API backend — it has no UI at the root path. The application you open in your browser is at http://localhost:5173.
